This script trains multiple Continuous-Time Recurrent Neural Networks (CTRNNs) across various hidden layer sizes on two NeuroGym decision-making tasks: Perceptual Decision Making (PDM) and Contextual Decision Making (CDM). Following training, it analyzes the hidden-state trajectories using Gaussian Partial Information Decomposition (PID).

Input Data Setup
### Option 1: Pre-generated NPZ Datasets
Provide paths to local `.npz` files generated by `data_generator.py`. Your directory structure should look like this:

```text
data/
├── pdm/
│   ├── train.npz
│   ├── val.npz
│   └── test_01.npz
└── cdm/
    ├── train.npz
    ├── val.npz
    └── test_01.npz
```

### Option 2: On-the-Fly Generation
If local dataset directories are omitted, the script automatically streams and generates synthetic trials directly from **NeuroGym**. No prior data prep is required.

---

## How to Run

### 1. Train a Single Hidden Size
```bash
python ctrnn_pid_both_tasks.py \
    --pdm_dir data/pdm \
    --cdm_dir data/cdm \
    --hidden_sizes 100
```

### 2. Train Multiple Hidden Sizes Sequentially
```bash
python ctrnn_pid_both_tasks.py \
    --pdm_dir data/pdm \
    --cdm_dir data/cdm \
    --hidden_sizes 20 50 100 200
```

### 3. Run Using On-the-Fly NeuroGym Data
```bash
python ctrnn_pid_both_tasks.py --hidden_sizes 100
```

---

## Command-Line Arguments

| Argument | Description | Default |
| :--- | :--- | :--- |
| `--hidden_sizes` | Space-separated list of hidden layer sizes to evaluate | `100` |
| `--pdm_dir` | Path to directory containing local PDM datasets | `None` |
| `--cdm_dir` | Path to directory containing local CDM datasets | `None` |
| `--num_epochs` | Maximum training epochs allowed per model | `50` |
| `--patience` | Epoch patience threshold for early stopping | `10` |
| `--lr` | Optimizer learning rate | `1e-3` |
| `--batch_size` | Training batch size | `2048` |
| `--n_test_trials` | Total test trials to generate (when using NeuroGym mode) | `2000` |
| `--n_bipartitions` | Number of random neuron bipartitions calculated for PID | `400` |

## Outputs & Directory Structure

All generated outputs are saved to the `results/` directory:

```text
results/
├── weights/
│   ├── ctrnn_pdm_h100.pth
│   ├── ctrnn_cdm_h100.pth
│   └── ...
├── training_curves_100.png
├── pid_pdm_100.png
├── pid_cdm_100.png
├── comparison_4panel_100.png
├── results_100.npz
└── ...
```

### 1. Model Weights (`weights/`)
Saves the best-performing model state dictionary based on validation accuracy (e.g., `ctrnn_pdm_h100.pth`).

### 2. Generated Figures
* **`training_curves_[size].png`**: Plots loss and accuracy tracking over epochs for both training and validation sets.
* **`pid_pdm_[size].png` & `pid_cdm_[size].png`**: Tracks Total Mutual Information, Synergy, Redundancy, and Unique Information across time.
* **`comparison_4panel_[size].png`**: Evaluates individual targets across tasks (PDM Coherence, PDM Decision, CDM Coherence, CDM Context).

### 3. Numerical Results (`.npz`)
Saves comprehensive run metrics to a single file per hidden size configuration.

```python
import numpy as np

# Example: Loading results for hidden size 100
data = np.load("results/results_100.npz")
print(data.files)
```

#### Inside the `.npz` File:

Every PID matrix array matches a 1D shape representing the trial sequence length: `(sequence_length,)`.

* **PID Core Metrics:** Replace `[task]` with `pdm` or `cdm`, and `[target]` with `coherence`, `decision`, or `context`.
  * `[task]_[target]_synergy`
  * `[task]_[target]_redundancy`
  * `[task]_[target]_unique1`
  * `[task]_[target]_unique2`
  * `[task]_[target]_mi_joint`
* **Timing Indicators:**
  * `[task]_stim_end`: Integer timestep marking the end of the stimulus presentation window.
* **Training History:**
  * `[task]_train_loss`, `[task]_val_loss`, `[task]_test_loss`
  * `[task]_train_acc`, `[task]_val_acc`, `[task]_test_acc`
